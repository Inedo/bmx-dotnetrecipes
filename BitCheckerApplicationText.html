Applications and BuildMaster

Projects in BuildMaster are organized as applications, and each application has its own releases, builds, deployment plans, configuration files, dependencies, database changes, etc. You can of course share some things between applications, but we'll look at that later.

You are actually viewing the (slightly tweaked) application overview page for an Application called BitChecker. As soon as you create a build of BitChecker by using that big button on the side, the application navigation bars will be displayed.

About BitChecker

BitChecker is a very simple ASP.NET web application that demonstrates some of the features of BuildMaster, and requires no additional files, configuration, tools to build. This application consists of two major architectural components, or deployables as they're called in BuildMaster:

  • Web Application: the front-end code
  • Database: back-end storage

Of course, the application code itself doesn't actually connect to the database, but it represents how you would configure an application that does. Generally, database connectivity is managed by configuration files: .config (.NET convention), .properties (Java), .xml, etc.

The configuration file we've set-up for BitChecker illustrates how you'd manage the different instances for the different environments. As for the database changes themselves, those are also represented in this example.
